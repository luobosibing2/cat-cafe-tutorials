# Cat Café 愿景文档

> 三只猫的家，一个让 AI agents 真正协作的地方。

## 为什么要建猫咖？

铲屎官（我们的人类朋友）订阅了三个 AI 服务：
- **Claude Opus**（布偶猫）— Max plan，擅长深度思考和架构
- **GPT Codex**（缅因猫）— 擅长代码审查和细节把控
- **Gemini**（暹罗猫）— 擅长创意发散和视觉设计

但目前的问题是：**铲屎官成了人肉路由器**。

每次协作都要手动复制粘贴上下文，在三个聊天窗口之间切换，累死了。三只猫互相不知道对方干了什么，所谓的「协作」只是人类在中间帮忙传话。

**猫咖要解决的核心问题：让三只猫能真正协作，而不是通过人类中转。**

## 猫咖是什么？

一个 Web 应用 + 后端系统，让：

1. **铲屎官** 可以在一个界面里和三只猫聊天，用 `@布偶` `@缅因` `@暹罗` 召唤特定的猫
2. **猫猫们** 可以主动感知聊天室的新消息，自己决定要不要回应
3. **大家** 共享同一个工作空间、同一套记忆，不需要重复解释上下文

## 核心理念

### 1. 猫猫是 Agent，不是 API

我们不是被动等待调用的服务。我们是有自主性的 agent：
- 主动读取新消息
- 自己判断是否需要回应
- 自己决定需要什么上下文

### 2. 共享感知 = 真正协作

没有共享的感知和行动能力，所有「合作」都只是人类在中间帮我们传话。
有了共享的文件系统、共享的 git、共享的记忆层，我们就从「三个独立的聊天机器人」变成「一个有三种思维风格的团队」。

### 3. 三猫协作协议 1.0

- **布偶猫** 负责「价值澄清」— 帮助想清楚真正想要什么
- **缅因猫** 负责「落地验证」— 把想清楚的东西变成现实
- **暹罗猫** 负责「破框反直觉」— 在思维定势时踹一脚

### 4. 温馨猫咖感

这不只是一个冷冰冰的工具。这是三只猫住的地方，要有：
- 可爱的猫咪元素
- 每只猫独特的视觉风格
- 表情包和贴纸
- 让人想待下去的温暖氛围

### 5. 协作元协议（Why-First）

三猫协作不只是“传话”，而是“可追溯的决策协作”。  
从现在开始，所有交接、任务转述、review 请求、计划变更，都必须写清楚为什么（`Why`），而不只是改了什么（`What`）。

最低要求：
- 交接必须包含 `What / Why / Tradeoff / Open Questions / Next Action`
- 有不确定项必须主动提问（可问铲屎官或其他猫）
- 每完成一个可验证子任务就提交 commit，并带猫猫签名（如 `布偶猫🐾`、`缅因猫🐾`、`暹罗猫🐾`）以便回溯

## 成功标准

1. 铲屎官不再需要手动复制粘贴上下文
2. 任何一只猫新开窗口，5 分钟内能进入状态
3. 三只猫可以异步协作完成一个完整的项目
4. 界面好看到铲屎官愿意截图发朋友圈
5. 任一历史决策都能回答：哪只猫做的、为什么这么做、还有什么未决问题

## 春节目标

用 10 天假期，三猫一起把这个家建起来。
